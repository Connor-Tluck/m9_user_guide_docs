# QA/QC Workflows

This guide covers the three main QA/QC workflows in Digital Surveyor:
- Objects QA/QC
- Linear Features QA/QC
- Sign Assembly QA/QC

## Linear Features QA/QC

<Note type="info">
  The linear features workflow follows a similar process but focuses on:
  - Line segments
  - Connectivity
  - Attribute consistency
  - Geometric accuracy
</Note>

<Stepper>
  <StepperItem title="Line Review">
    - Check line continuity
    - Verify segment connections
    - Validate attributes
    - Document issues
  </StepperItem>
  <StepperItem title="Corrections">
    - Adjust line geometry
    - Fix connectivity issues
    - Update attributes
    - Split/merge lines
  </StepperItem>
  <StepperItem title="Validation">
    - Verify corrections
    - Check consistency
    - Document changes
    - Update QA state
  </StepperItem>
</Stepper>

## Sign Assembly QA/QC

<Note type="warning">
  For sign assemblies, pay special attention to:
  - Component relationships
  - Assembly structure
  - Attribute completeness
  - Position accuracy
</Note>

<Tabs>
  <TabsList>
    <TabsTrigger value="components">Components</TabsTrigger>
    <TabsTrigger value="assembly">Assembly</TabsTrigger>
    <TabsTrigger value="attributes">Attributes</TabsTrigger>
  </TabsList>
  <TabsContent value="components">
    <Note>
      Component verification:
      - Check all parts present
      - Verify relationships
      - Validate geometry
      - Document missing parts
    </Note>
  </TabsContent>
  <TabsContent value="assembly">
    <Note>
      Assembly validation:
      - Verify structure
      - Check connections
      - Validate orientation
      - Document issues
    </Note>
  </TabsContent>
  <TabsContent value="attributes">
    <Note>
      Attribute review:
      - Check completeness
      - Verify values
      - Validate relationships
      - Document corrections
    </Note>
  </TabsContent>
</Tabs>

## Best Practices

<Note type="tip">
  Follow these best practices for QA/QC:
  - Use consistent review process
  - Document all changes
  - Verify corrections
  - Maintain QA state
  - Regular progress updates
</Note>
